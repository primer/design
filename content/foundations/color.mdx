### Color roles or highlights

#### Global roles
Special group of highlights to create boundaries inside a layout or highlight an area of content without any specific meaning.

| Variable | Role |
| -------- | -------- |
| `neutral-` | Highlights content without any added meaning |
| `accent-` | Highlights important interactive elements in the interface |
| `success-` | Expresses completion or positive outcome of a task|
| `attention-` | Warns of pending tasks or highlights active content  |
| `severe-` | Mostly used when there's more than 3 levels of states like heatmaps |
| `danger-` | Informs of error or negative messaging |

#### GitHub-specific roles

| Variable | Role |
| -------- | -------- |
| `done-` | Completion color for productivity and code review workflows |
| `sponsors-` | Color suite for all sponsors-related content |
